# Contributing

感谢你愿意给 PathoScope 添砖加瓦——最好是添功能，不是添新的黄色满屏。

## 推荐流程

1. 先开 Issue 描述问题、切片格式、macOS 与芯片型号。
2. Fork 仓库并从 `main` 创建功能分支。
3. 不要提交真实患者切片、患者姓名、绝对路径、缓存或访问凭据。
4. 修改核心逻辑时补充测试，并运行：

```bash
swift test -c release
swift build -c release
```

5. PR 请写清楚：改了什么、为什么、怎样验证、还剩什么风险。

## 图像问题的最小复现

优先提供脱敏后的元数据、格式、level、tile 坐标与预期/实际行为。若必须共享切片，请使用你有权分享的匿名样本，并通过双方同意的私密渠道；不要直接塞进公开 Issue。GitHub 不是数字病理的公共冰箱。
